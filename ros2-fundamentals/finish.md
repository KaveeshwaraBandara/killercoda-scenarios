# That's the ROS 2 core

In one session on a real Linux machine you:

- **inspected** a live graph (`node list`, `node info`, `topic info`, `interface show`, `echo`, `hz`)
- **injected** messages by hand with `topic pub` — and fooled a professional node into obeying you
- **wrote** a publisher, and had someone else's node consume it
- **wrote** a service server, and called it
- **tuned** a running node from outside with parameters

That's the toolkit every ROS 2 developer uses daily. Nothing to clean up — this machine destroys itself when your session ends.
